The “var” thing

Fri Sep 4, 2009 by

I’ve come to really enjoy using the “var” keyword in C#. Although it’s been written about many times, I want to clarify what it is. C# is a strongly-typed language, which means that every variable you declare must have a specified type. You must say it’s an integer or a string or a Thing. “var” does not change that. “var” looks a lot like the var in Javascript.
