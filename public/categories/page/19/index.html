Solving the n+1 problem in Linq to SQL, using projection

Mon Oct 12, 2009 by

Rick Strahl has a nice solution for mitigating the n+1 problem in Linq to SQL. The n+1 problem is a situation where, as you iterate over a set of query results in a list view, one or more additional database queries are fired for each row in the list. Since database calls are relatively expensive, you want to keep these to a minimum. Things like Linq to SQL make this an easy mistake to make. Read more →
