Does null ever equal anything?

Yesterday’s discussion of nulls got me to thinking about this common bit of syntax in C#: [code:c#]if (myObject == null) { … [/code] But here’s the thing: null is not data. It’s metadata. That is, it describes the state of a variable, not its contents. Null indicates the absence of a value. So the above code says, is myObject’s value equal to not having a value? Seems strange.
